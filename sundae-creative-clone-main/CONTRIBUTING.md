# Contributing to Sundae Creative Clone

Thank you for your interest in contributing to this project! Here's how you can help.

## Getting Started

1. Fork the repository
2. Clone your fork:
   ```bash
   git clone https://github.com/YOUR_USERNAME/sundae-creative-clone.git
   cd sundae-creative-clone
   ```
3. Create a new branch:
   ```bash
   git checkout -b feature/your-feature-name
   ```
4. Install dependencies:
   ```bash
   npm install
   ```

## Development Workflow

1. Run the development server:
   ```bash
   npm run dev
   ```
2. Make your changes
3. Test your changes in the browser
4. Run linting:
   ```bash
   npm run lint
   ```

## Code Style

- Follow the existing code style
- Use Prettier for formatting: `npm run format` (if configured)
- Write meaningful commit messages
- Use ESLint to catch issues: `npm run lint`

## Making Changes

### For Bug Fixes
1. Create a branch from `main`
2. Fix the bug
3. Add tests if applicable
4. Submit a pull request with a clear description

### For New Features
1. Create a feature branch
2. Implement the feature
3. Test thoroughly
4. Submit a pull request with:
   - Clear description of the feature
   - Why it's needed
   - How to test it

## Commit Messages

Use clear and descriptive commit messages:
- ✅ Good: "Add smooth scroll animation to section 2"
- ❌ Bad: "Fix stuff"

## Pull Request Process

1. Update the CHANGELOG.md with your changes
2. Ensure all tests pass
3. Run `npm run lint` to check code style
4. Provide a detailed description of your changes
5. Link any related issues

## Reporting Issues

When reporting bugs, please include:
- Description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Browser and OS information

## Questions?

Feel free to open an issue for questions or discussions about the project.

Thank you for contributing! 🎉
